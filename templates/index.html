Migrate to another instance

Move all your apps and data to another OpenHost instance. Install the backup app on both instances, then generate an API token on the destination. Your data will be transferred and your applications will be restarted on the other side.

Important details before migrating
What happens on the destination: All running apps on the destination will be stopped, and existing app data for migrated apps will be deleted before new data is written. Apps not included in the migration will be stopped during the transfer and restarted afterward.

Tip: It is a good idea to stop applications on this instance before migrating, especially databases or apps that might not transfer correctly if they are actively writing data. Stopping is not required, but it reduces the risk of inconsistent data.

Not migrated: API tokens, owner password, and other system-level settings are not transferred. You will need to set these up separately on the destination instance.
