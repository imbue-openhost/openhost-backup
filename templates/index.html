Repository

The backup app stores snapshots in a restic repository. By default it points at a local path inside the backup app's own data directory. You can change this to any restic-supported backend. For AWS S3, use the region-specific endpoint matching your bucket's region, e.g. s3:s3.us-west-2.amazonaws.com/bucket/path (using the generic s3.amazonaws.com endpoint works but adds an extra redirect on every request and can fail signature checks if AWS_DEFAULT_REGION doesn't match the bucket). Other examples: b2:bucket:path, sftp:user@host:/path. Changing the repo URL does not migrate existing data. The repo password is required to read or write the repo — losing it makes backups unrecoverable.

Backend credentials

Credentials forwarded to restic as environment variables. Fill these in when pointing at a remote backend (S3, B2, Azure, etc.). Values are stored in config.json alongside the repo password. Leave a field blank to leave the current value unchanged; enter a single space to clear it.

Pre-migration tools

Stop all applications and fix potential file ownership issues. If you don't know what that means, you should do this. Migrations without stopping applications should only be performed if you have a specific reason to do so.

Migrate to another instance

Move all your apps and data to another OpenHost instance. Install the backup app on both instances, then generate an API token on the destination. Your data will be transferred and your applications will be restarted on the other side.

Important details before migrating
This instance: Not changed. Migration copies your data to the destination. Nothing on this instance is modified or deleted.

Destination: All running apps will be stopped, and existing app data for migrated apps will be deleted before new data is written. Apps not included in the migration will be stopped during the transfer and restarted afterward.

Tip: It is a good idea to stop applications on this instance before migrating, especially databases or apps that might not transfer correctly if they are actively writing data. Stopping is not required, but it reduces the risk of inconsistent data.

Not migrated: API tokens, owner password, and other system-level settings are not transferred.
